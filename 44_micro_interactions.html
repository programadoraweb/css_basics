    the communications between the user and the system will be much better when the system gives to user small feedbacks through micro-interactions during the communication process.

    you can trigger micro-interactions with pseudo clases or clases changes

    Elements that micro-interactions should apply:

    • font-size
    • background-color
    • width
    • left

    Elements that micro-interactions should NOT apply:

    • change font-family
    • position:from absolute to relative
    • change display ie. block to inline
    • left

    Micro-interactions to solve problems:

    1. Orientation on the page

      2. where are you now in the page, where can you go and from where did you come

    2. Functional Change

      3. what does this element do now. ie. switch button

    3. New Elements

      4. What changed? ie. new elements could appear smoothly.

    4. Highlight

      5. catch the user s attention to what is important on this page

    5. Visual Feedback

      6. the system understood the info that you gave ie. input data

    6. System Status

      7. to understand that the system is processing info, ie. progress bar

    for performance reasons better makes interactions with rotation, opacity, scale & positions. Other properties could give a bad performance (bellow 60fps), as other properties could reprint the element on the screen. Rotatio, opacity, scale & positions will just change the element they're focusing in.
